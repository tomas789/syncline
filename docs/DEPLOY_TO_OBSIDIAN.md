# Deploying Syncline to the Official Obsidian Plugin List

## Overview

This guide explains how to submit the Syncline plugin to Obsidian's official community plugin directory.

## Prerequisites

1. **GitHub Account** - Required for hosting the repository
2. **Obsidian Forum Account** - Required for the submission thread
3. **Plugin Manifest** - Must be complete and accurate
4. **Tested Plugin** - Should work reliably in Obsidian

## Required Files

Ensure these files exist:

- `manifest.json` - Plugin metadata (in root)
- `obsidian-plugin/main.js` - Compiled plugin (generated by `npm run build`)
- `obsidian-plugin/styles.css` - Optional styles for the plugin
- `README.md` - Documentation for users

### manifest.json Example

```json
{
  "id": "syncline-obsidian",
  "name": "Syncline",
  "version": "1.0.0",
  "minAppVersion": "0.15.0",
  "description": "Real-time collaborative editing for Obsidian vaults using CRDTs",
  "author": "Your Name",
  "authorUrl": "https://github.com/yourusername",
  "isDesktopOnly": false
}
```

### 1. Prepare the Release

With the new monorepo structure, releases are managed directly from the main [syncline](https://github.com/tomas789/syncline) repository.

The automated GitHub Action handles building the plugin and native binaries whenever you publish a release.

### 2. Create a GitHub Release

1. Go to the [syncline](https://github.com/tomas789/syncline) repository on GitHub.
2. Click "Releases" → "Draft a new release".
3. Tag: `0.1.0` (must match `manifest.json` version).
4. Title: `Syncline v0.1.0`.
5. Click "Publish release".
6. The **Release** GitHub Action will automatically:
   - Build the Rust WASM module.
   - Build the Obsidian plugin (`main.js`).
   - Compile Linux amd64 binaries for the server and client.
   - Attach all assets to the release.

### 3. Submit to Community Plugins

1. Fork the [obsidian-releases](https://github.com/obsidianmd/obsidian-releases) repository

2. Edit `community-plugins.json`, add your plugin:

```json
{
  "id": "syncline-obsidian",
  "name": "Syncline",
  "author": "tomas789",
  "description": "Real-time collaborative editing for Obsidian vaults using CRDTs",
  "repo": "tomas789/syncline",
  "branch": "main"
}
```

3. Create a Pull Request with:
   - Title: "Add Syncline plugin"
   - Description: Brief overview of the plugin

4. Wait for review by Obsidian team

### 4. Forum Announcement (Optional but Recommended)

Post in the [Plugin Releases](https://forum.obsidian.md/c/plugin-releases/9) category:

- Title: `[Release] Syncline - Real-time collaborative editing`
- Content: Describe features, installation, usage, and link to GitHub

## Version Updates

For each new version:

1. Update `manifest.json` version number
2. Build the plugin: `npm run build`
3. Commit and push changes
4. Create a new GitHub Release with updated assets
5. The community plugin list will auto-update

## Important Notes

### WASM Handling

Since this plugin uses WebAssembly:

- The WASM binary is embedded as base64 in `main.js` (via rollup-plugin-base64)
- No separate `.wasm` file needed in the release
- Total file size will be larger (~600KB+)

### Security Considerations

- Never commit API keys or secrets
- The `syncline.db` file is user data, not part of the plugin
- Server component is separate and not part of the plugin release

### Testing Before Release (Desktop)

1. Install the built `main.js` in a test vault
2. Verify all features work correctly
3. Check the console for errors
4. Test with the server running

### Testing on iPhone (iOS)

To test the Syncline plugin locally on your iPhone before publication, you need to load the plugin into your iOS vault.

**Method 1: iCloud Drive (Recommended)**

1. Ensure your test vault on your iPhone is stored in iCloud Drive.
2. From your Mac, open Finder and navigate to `iCloud Drive/Obsidian/[Your Vault]`.
3. Press `Cmd + Shift + .` to show hidden files and open the `.obsidian` folder.
4. Navigate to `plugins` and create a new folder called `syncline-obsidian` (matching your `manifest.json` ID).
5. Copy your built `main.js`, `manifest.json`, and `styles.css` (if any) into this folder.
6. Wait for iCloud to sync the files to your iPhone.
7. Open Obsidian on your iPhone, go to **Settings > Community plugins**, turn off "Safe Mode" if it's on, and enable the plugin.

**Method 2: Obsidian Sync**

1. Install and enable the plugin in your desktop Obsidian vault.
2. Make sure Obsidian Sync is configured to sync "Community plugins" and "Installed community plugins".
3. Wait for Obsidian Sync to push the changes, then enable the plugin from the iPhone app settings.

**Debugging on iPhone**

1. Connect your iPhone to your Mac via USB.
2. On your iPhone, go to **Settings > Safari > Advanced** and turn on **Web Inspector**.
3. Open Obsidian on your iPhone.
4. Open Safari on your Mac, go to the **Develop** menu, select your iPhone, and click the Obsidian web view. This will open Web Inspector for the mobile app, allowing you to view console logs and inspect elements.
   _(Note: Mobile platforms do not support Node.js or Electron APIs. Because Syncline uses WebAssembly, verify your WASM loads efficiently on mobile Safari.)_
